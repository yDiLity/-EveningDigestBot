# PRD: Бот «Итоги дня» (Daily Digest Bot)

**Версия:** 1.0.0  
**Статус:** Готов к разработке  
**Тип продукта:** Telegram-бот  
**Целевая платформа:** Telegram (личные сообщения + супергруппы)

---

## 1. Концепция

Бот собирает от участников команды короткие отчеты о том, что они сделали за день (еда, активность, учеба, работа), а вечером публикует их в общую группу.

**Ключевая фишка:** бот моментально показывает красивый черновик после каждого сообщения, а вечером постит отдельные сообщения на каждого человека + сравнительный итог только по тем метрикам, которые указали минимум 2 участника.

---

## 2. Цели

| Цель | Для кого |
|------|----------|
| Мотивировать участников фиксировать свои достижения | Участники |
| Получать структурированный, читаемый дайджест без хаоса в чате | Руководитель / команда |
| Видеть, кто в какой метрике лидирует (без спама) | Все участники |

---

## 3. Пользовательские сценарии

### Сценарий А: Обычный день участника

Утром участник пишет боту: «На завтрак овсянка с бананом»

Бот отвечает черновиком:
```
✅ Добавлено. Текущий черновик: завтрак овсянка с бананом.
```

Днем участник пишет: «Прошел 8000 шагов»

Бот обновляет черновик:
```
✅ +8000 шагов. Текущий черновик: завтрак овсянка, шаги 8000.
```

Вечером участник дописывает: «Прочитал 30 страниц»

В 21:00 бот автоматически публикует в группу:
- Персональный пост участника
- (если есть) сравнительный блок по шагам и страницам с другими

### Сценарий Б: Участник хочет отправить отчет сейчас

Пишет `/sendnow` — бот отправляет накопленный черновик в группу немедленно (а не ждет 21:00)

### Сценарий В: Участник ошибся

Пишет `/undo` — бот удаляет последнее добавленное действие

Пишет `/mydigest` — бот показывает полный текущий черновик

### Сценарий Г: Участник пропускает день

Пишет `/skip` — бот отмечает, что сегодня участник не участвует (не ждет от него отчета)

## 4. Функциональные требования

### FR-1: Прием сообщений

| Требование | Детали |
|-------------|--------|
| Формат ввода | Любой текст (естественный язык) |
| Каналы ввода | Только личные сообщения боту |
| Ограничение | 1 отчет в день от пользователя (но внутри — сколько угодно сообщений, они накапливаются) |
| Пропуск дня | Команда `/skip` — бот не ждет отчет от этого пользователя сегодня |

### FR-2: Парсинг и извлечение метрик

Бот распознает и извлекает из текста:

| Метрика | Ключевые слова | Единица измерения | Приведение |
|---------|---------------|-------------------|-------------|
| Шаги | шаг, шагов, шаги, прошел, км, километр | шаги, км | 1 км = 1300 шагов |
| Вода | литр, воды, выпил, стакан, кружка, мл | литры | 1 стакан = 0.25 л |
| Страницы | страниц, стр, прочитал, главу | страницы | — |
| Тренировка | присел, отжался, жим, подход, тренировка | повторы / кг | сохраняется как текст + число |
| Еда | завтрак, обед, ужин, съел, поел | текст (не сравнивается) | — |
| Работа | сделал, запушил, закончил, отчет, задачу | текст (не сравнивается) | — |

Если метрика не распознана — сохраняется как текст в категорию «Другое».

### FR-3: Мгновенный черновик (после КАЖДОГО сообщения)

После каждого сообщения пользователя бот присылает:

```
✅ Добавлено к вашему отчету за сегодня

📋 Текущий черновик:
🍳 Завтрак: овсянка с бананом
🚶‍♂️ Шаги: 8 000
📖 Страницы: 30

🕒 Автоотправка в группу в 21:00
/sendnow — отправить сейчас
/undo — удалить последнее
```

### FR-4: Хранение данных

| Данные | Срок хранения |
|---------|---------------|
| Черновик текущего дня | до 21:00 или до /sendnow |
| История отчетов | 30 дней (для статистики) |
| Личные данные пользователей | бессрочно (user_id, username, имя) |

### FR-5: Публикация в группу — ФОРМАТ

#### 5.1 Персональные посты (всегда)

Публикуется отдельное сообщение на каждого участника, кто прислал отчет за день.

Формат:

```
👤 Имя участника, ваш день:
🚶‍♂️ Шаги: 8 742
💧 Вода: 1.5 л
📖 Страницы: 20
🍳 Завтрак: овсянка
💼 Работа: запушил фичу логина
🕒 Отчитался в 20:15
```

**Правила:**
- Порядок метрик: числовые → текстовые → время
- Если метрика не указана — строка отсутствует
- Имя берется из Telegram (first_name или username)

#### 5.2 Сравнительный итог (только если метрику указали ≥2 человека)

Формат:

```
🏆 Сравнение за день:

🚶‍♂️ Шаги:
🥇 Марина — 12 300
🥈 Алексей — 8 742

💧 Вода:
🥇 Алексей — 1.5 л
🥈 Денис — 0.8 л
```

**Критическое правило:**
Если метрику указал только 1 человек — блок по этой метрике полностью отсутствует.

**Порядок вывода блоков:**
По убыванию количества участников, указавших метрику (сначала самые популярные).

#### 5.3 Если нет ни одной метрики с ≥2 уча��тниками

Сравнительный итог не публикуется (только персональные посты).

### FR-6: Расписание

| Событие | Время |
|---------|-------|
| Автоматическая публикация | 21:00 по времени группы |
| Напоминание отправить отчет | 18:00 (только тем, кто еще не отправил и не поставил /skip) |
| Сброс черновиков | 00:01 (новый день) |

### FR-7: Команды

| Команда | Описание | Доступ |
|---------|----------|--------|
| /start | Приветствие, инструкция | Все |
| /sendnow | Отправить текущий черновик в группу немедленно | Все |
| /mydigest | Показать текущий черновик (в ЛС) | Все |
| /undo | Удалить последнее добавленное действие | Все |
| /skip | Пропустить сегодня (бот не ждет отчета) | Все |
| /leaderboard | Текущая таблица лидеров по шагам (в ЛС) | Все |
| /post_all | Принудительная публикация (сбор всех отчетов) | Только админ группы |

### FR-8: Обработка ошибок

| Ситуация | Ответ бота |
|----------|------------|
| Не распознал ничего | «Не удалось распознать. Просто напишите: „прошел 5000 шагов“, „выпил 1л воды“ или „съел пиццу“» |
| Пользователь пишет в группу (не в ЛС) | «Пожалуйста, пишите мне в личные сообщения: @bot_username» |
| Пользователь пытается отправить 2 отчета за день после /sendnow | «Вы уже отправили отчет за сегодня. Завтра начнем новый» |
| В группу нечего публиковать | (ничего не отправлять, просто молчать) |

## 6. Логика для разработчика (алгоритмы)

### 6.1 Алгоритм публикации в 21:00

```
1. Получить всех пользователей, которые:
   - отправляли сообщения сегодня
   - не использовали /skip

2. Для каждого:
   - Собрать черновик
   - Опубликовать отдельное сообщение в группе

3. Собрать все метрики по всем пользователям:
   - steps: {user: value}
   - water: {user: value}
   - pages: {user: value}

4. Для каждой метрики:
   - Отфильтровать None (кто не указал)
   - Если len(filtered) >= 2 → добавить в блок сравнения
   - Отсортировать по убыванию

5. Если есть блоки сравнения → опубликовать их (одним сообщением)
6. Если нет блоков сравнения → ничего не публиковать (только персональные)
```

### 6.2 Алгоритм приведения единиц

```python
def normalize_steps(text: str, value: float, unit: str) -> int:
    if unit == "km":
        return int(value * 1300)
    if unit == "steps":
        return int(value)
    return None  # не распознано
```

### 6.3 Алгоритм распознавания метрик (простой)

Для MVP используется regex + словарь ключевых слов:

```python
METRICS = {
    "steps": (r"(прошел|шагов|шаги|км|километр)\s*(\d+)", "steps_or_km"),
    "water": (r"(выпил|литр|стакан|кружка|воды)\s*(\d+\.?\d*)", "liters"),
    "pages": (r"(прочитал|страниц|стр|главу)\s*(\d+)", "pages"),
}
```

(В следующих версиях — замена на YandexGPT / GigaChat)

## 8. Риски и митигация

| Риск | Вероятность | Решение |
|------|-------------|---------|
| Пользователь пишет сложные предложения («Я сегодня прошел где-то около 5 километров, наверное») | Средняя | Regex найдет «5 километров», остальное в other_text |
| LLM не нужен для MVP | Низкая | Используем regex. LLM добавим во v2 |
| Бот банит за спам в группе (слишком много сообщений) | Низкая | Персональных постов = количество участников (обычно 3-10) |
| Пользователь случайно пишет в группу, а не боту | Средняя | Бот отвечает в группе: «Напиши мне в ЛС» |

## 11. Критерии приемки (Definition of Done)

- [ ] Пользователь пишет «прошел 5000 шагов» → бот показывает черновик с шагами
- [ ] Пользователь пишет 3 сообщения за день → черновик накапливается
- [ ] В 21:00 бот постит отдельные сообщения на каждого
- [ ] Шаги указали 2 человека → есть блок сравнения шагов
- [ ] Страницы указал 1 человек → нет блока сравнения страниц
- [ ] Команда `/sendnow` публикует немедленно
- [ ] Команда `/undo` удаляет последнее действие
- [ ] Бот не падает при нераспознанном сообщении
